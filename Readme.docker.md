# 도시이야기 Docker 실행 가이드 (Readme.docker.md)

이 문서는 "도시이야기" 웹 애플리케이션을 Docker 및 Docker Compose 환경에서 패키징하고 구동하는 방법을 안내합니다.

---

## 1. 사전 준비사항
- 시스템에 **Docker Desktop** 또는 Docker Engine이 설치되어 활성화되어 있어야 합니다.

---

## 2. 프로젝트 컨테이너 구조
본 프로젝트는 Docker Compose를 통해 2개의 컨테이너 서비스로 구동됩니다.
1. **web (`city_story_web`)**:
   - `php:8.2-apache` 기반으로 빌드된 웹 서버입니다.
   - MySQL 연동을 위해 `mysqli` PHP 확장이 사전에 활성화되어 있습니다.
   - 호스트의 소스 코드 폴더 전체가 실시간 마운트되어 코드 변경이 즉시 반영됩니다.
2. **db (`city_story_db`)**:
   - `mysql:8.0` 데이터베이스 서비스입니다.
   - 컨테이너가 처음 실행될 때 `table/userinfo.tbl.sql` 스크립트를 사용하여 `userinfo` 테이블을 자동으로 빌드합니다.

---

## 3. 실행 명령어 가이드

### 3.1 컨테이너 빌드 및 백그라운드 실행
프로젝트 루트 폴더에서 아래 명령어를 실행하여 웹 서버와 DB 컨테이너를 함께 구동합니다.
```bash
docker compose up -d --build
```
- `-d`: 백그라운드 실행 옵션
- `--build`: Dockerfile 변경사항을 반영하여 이미지를 새로 빌드하는 옵션

### 3.2 작동 상태 확인
컨테이너의 동작 유무 및 포트 상태를 파악합니다.
```bash
docker compose ps
```

### 3.3 웹 애플리케이션 접속
브라우저를 열고 아래 주소로 접속해 테스트를 진행합니다.
- **웹 페이지 주소**: [http://localhost:8080](http://localhost:8080)
- **DB 포트**: 호스트의 `3306` 포트로 매핑되어 있어, 외부 DB 클라이언트(DBeaver, HeidiSQL 등)로 가상 DB에 직접 연결할 수 있습니다. (사용자: `root`, 암호: `rootpassword`)

### 3.4 실시간 컨테이너 로그 모니터링
접속 로그나 PHP 오류 기록을 확인하고 싶을 때 사용합니다.
```bash
docker compose logs -f
```

### 3.5 서비스 정지 및 컨테이너 삭제
서비스 동작을 완전히 중단하고 컨테이너를 제거합니다.
```bash
docker compose down
```

### 3.6 데이터 초기화 및 완전 삭제
데이터베이스 볼륨까지 전부 파괴하고 처음부터 깨끗한 상태로 재시작하고 싶을 때 사용합니다.
```bash
docker compose down -v
```
- `-v`: 데이터베이스 데이터가 담긴 볼륨(`db_data`)을 삭제하여 DB를 완전 초기화시킵니다.

---

## 4. 문제 해결 및 주의사항

> [!WARNING]
> **포트 충돌 해결 방법 (Port 3306 / 8080)**
> - 로컬 환경에 이미 **XAMPP MySQL** 서비스나 다른 데이터베이스가 실행 중인 경우, 호스트 포트 `3306` 충돌로 인해 컨테이너 시작에 실패할 수 있습니다.
> - **해결법**: 로컬 MySQL 서비스를 완전히 중지한 후 기동하거나, `docker-compose.yml` 파일에서 `db` 서비스의 `ports` 항목을 `"3307:3306"` 등으로 다르게 수정하여 충돌을 회피하십시오.
