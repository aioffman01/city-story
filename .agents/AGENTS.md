# Project Rules

이 프로젝트는 작업할 때 반드시 다음 규칙을 엄격히 준수해야 합니다. 



1. 한국어로 무조건 답할것 : 

* 모든 생각과 사용자 응답은 자연스러운 한국어로 작성할 것 
* &nbsp;변수명이나 코드내의 영문은 그대로 두되, 주석이나 설명은 한국어를 사용 할것 



2\. 안정성이 우선 ( 사전 동의 필수 ) :

* 기존의 잘 돌아가는 코드를 대폭 수정하거나 파일을 삭제하기 전에는 사용자에게 "이렇게 변경해도 될까요?"라고 물어보고 승인을 얻은 후 진행 할 것 



3\. 단계별 계획 수립 :

* 복작한 기능을 구현할때는 코드를 짜기전에 먼저 구현계획을 마크다운 아티팩트로 정리하여 사용자에게 보여 줄것 



4\. 친절하고 충분한 코드 설명(교육용) : 

* 사용자는 전문 개발자가 아닌 코딩 초보자(교육자)입니다. 코드를 작성하거나 수정할 때마다 단순한 코드만 던져 주지 말고, 각 로직이 어떤 역할을 하는지 한글 주석과 함께 설명해 줄것 



&nbsp;5. PHP 프로젝트의 코딩 규칙은 루트 폴더에 위치한 PHP.md 화일에 적혀 있는 규칙을 따른다. 



* **디자인 문서화 규칙**: 본 프로젝트 진행 중 웹 디자인 요소가 변경되거나 추가 디자인 패턴(컬러 토큰, 입력 폼 컴포넌트, 폰트, 호버 인터랙션, 새로운 페이지 테마 등)이 신규로 도입되는 경우, 루트 폴더에 위치한 [DESIGN.md](file:///c:/Users/redof/Documents/Antigravity%20Project/DESIGN.md) 파일에 해당 세부 사양 정보를 상세히 업데이트하며 작업해야 합니다.
